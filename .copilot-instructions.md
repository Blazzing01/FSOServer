Project overview:
1. The server and client are using Lidgern as the networking Library and the server is completely authoritative.
2. This is a game server written in C# .net for a 2d RPG Game In a similar style to Runescape.
3. The game has a single world as defined by the /World/GameWorld.cs class
4. The world is composed of "Maps" Each map is composed of 32x32 pixel Tiles and the map is 50 tiles wide and 40 Tiles tall.
5. Maps are composed of 5 "Layers" with layer 2 being the "Collision Layer" which defines where players can and can not walk. The other layers are used for visual purposes.

Networking flow:
1. Client sends a packet to the server. The packets first variable is an unsigned short that represents the ID of the packet type.
2. In the ServerManager.cs class, the RunAsync() method is called and contains a loop that continously checks for packets and processes them.
3. The SortPacket() Method is called which reads the first Ushort to find the type of incoming packet.
4. Once the packet is identified, we use the PacketFactory.cs class to return the command to run based on the packet ID
5. The command is ran and the server state is updated. Such as "Move character"  "Send Chat message"  etc.

Gameplay flow:
1. The player controls a character and the character is clamped to the middle of each tile.
2. When a player moves, over a period of time the character will move to the next tile. During this period they can not change directions or move again until the character arrives at the middle of the next tile.
3. When the character attempts to move past the map bounds, the character should transfer to the next map in the series in that direction and load the map from the server.
4. Players level up skills such as Mining, blacksmithing, woodcutting etc.
5. Tab is the attack button, CTRL is the interact button (Activating Switches and picking up items)
